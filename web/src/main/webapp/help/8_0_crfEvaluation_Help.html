8. CRF Evaluation Help

CRF evaluation allows a new user role "Evaluator" to only access the CRF(s) to be evaluated without being able to access any other data in a study. Just like a CRC, this role is allowed to edit/view/print the CRFs to be evaluated.

CRF Evaluation has 2 system-level properties:

  • Allow CRF Evaluation: Allows the Evaluator user role to appear when defining users or changing roles of study level users.
  • Evaluate with Context: Displays additional information on the reading page, such as Subject ID and the Event Name.

The system-level properties can be overridden at the study level.

When a CRF is determined to be part of the CRF Evaluation page, a setting is turned on in the Edit Study Event Definition page. If the box "Evaluate CRF" is checked, the CRF appears in the CRF Evaluation page.

This work is carried out by a new Evaluator Role. This role can be defined at the Study or Site level. As a Study level role, the Evaluator has access to all sites under a study. As a site-level role, the Evaluator can only access CRFs to be evaluated in specific sites. The Evaluator does not see any other study data, including queries, unless they generate them themselves.
