5.2.1 Calendared Event Definitions Help

This pop-up page allows you to view all the Study Event Parameters that are a part of your current study, especially Calendared Events. Note that there are a number of parameters that appear when your Event is Calendared. They are the following:

Event Name: This cell displays the Study Event Name.
Event Type: This cell indicates the Study Event Type, Calendared, Scheduled, Common or Unscheduled.
Reference: This cell indicates whether or not a Calendared Event is a reference visit. Typically a reference visit is a baseline event which triggers other events to be scheduled. Completing a reference visit will result in the non-reference Calendared Events to be scheduled for a Subject, according to the rest of the parameters.
Day Min: This cell indicates the minimum number of days that should pass before a Calendared Event is completed. This is calculated as the Date a reference visit is completed + number of days in Day Min. If an Event is completed before its Day Min, a Query will be generated and the Calendared Icon next to the Subject in the Subject Matrix will turn red.
Day Max: This cell indicates the maximum number of days that can pass until a Calendared Event should be completed. This is also calculated as the Date a reference visit is completed + number of days in Day Min. If an Event is completed after its Day Max, a Query will be generated and the Calendared Icon next to the Subject in the Subject Matrix will turn red.
Day Schedule: This cell indicates the day an Event will be auto-scheduled after the completion of a reference event. Naturally, the Day Scheduled needs to be between Day Min and Day Max.
Day Email: This cell indicates the day that a reminder email will be sent out about the Event. This is calculated in the same way as Day Min and Day Max. The Day Email should come before Day Scheduled.
