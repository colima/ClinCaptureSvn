About Discrepancy Notes
ClinCapture provides discrepancy management capabilities for verifying the accuracy and completeness of data in a clinical trial, to address incomplete or inaccurate data, provide tracking and auditing on changes to the data, and to track accuracy and completeness for studies, sites, users, and CRFs. Discrepancy Management may be turned on or off for each study as part of the study configuration.

A discrepancy note is always associated with a particular subject-specific data element (or item). This may be an element that is entered when adding the subject record, such as date of birth, or elements created when recording a patient encounter, such as the date of a study event or an item in a CRF.

Discrepancy Notes can be 'threaded' with a parent and child notes linked to the parent - this allows recording of queries and review actions on a single note over time, ending with the creation of a 'Closed' note. Nested (multi-level) threading is not currently supported. Multiple discrepancy note threads may exist for a single data element.

Notes & Discrepancies Page
The Discrepancy Page allows a Study Manager to view and resolve discrepancy notes that are generated during the course of data capture and verification. The screen will list all discrepancy notes for a given Study, and can take you to a more detailed view for each note, or through a workflow to resolve open notes. Where relevant, entity names will link to data dictionary popups to provide more information on which data element the note refers to.
You can use the Filter by status or the Filter by note type to search for specific notes & discrepancies. Just click on the flag(s) and/or the note type(s) you want to filter. You can reset the filter by clicking on the links Clear status filter and/or Clear type filter which will appear if you have set a filter.

In the top of the table you can use the green arrows to navigate through the pages. If you have less than 15 notes in your study the arrows are greyed out, which means that there is only one page to display.
With the drop-down menu next to the arrows you can choose how many notes should be displayed on one page (15, 25, or 50).

With the drop-down menu, Select An Event in the top of the subject list, you can filter the subjects by an event. You can also filter by entering free text or by selecting a pre-defined choice from the drop-down menu, in the grey boxes underneath each name header. To apply the filter select the Apply Filter link.
To reset the filter click on the Clear Filter link next to it.

Note: You may click the Download Discrepancy Notes for all Subjects icon at the top of the table to download all notes in comma separated values format or in portable document format (to be chosen in a popup window). This report will show all discrepancy notes, will not be paginated, and will be formatted for printing.
You can also download all notes of a specific Study Subject. Therefore, click on the appropriate symbol under Actions next to the subject.

To view a list of all discrepancy notes with more details click on the List of Notes and Discrepancies icon located in the upper right corner of the table.

You are also provided with the link Show summary statistics (above the matrix) to get an overview about all notes and discrepancies in your current study. Note that each number is a hotlink to those types of Notes in the table, so you can click on the link and immediately go to all New Queries, for example.

Creating Discrepancy Notes
Discrepancy Notes are created by clicking on a flag icon next to the corresponding data element in a form such as an 'Enroll Subject' form or a CRF. Anywhere you see this flag, you may add a discrepancy note to that item. If a discrepancy note already exists, you may view the existing note(s) or add an additional one. Javascript must be enabled and popup blockers should be disabled for discrepancy notes to work correctly. Clicking the flag icon will call a popup window, in which you can enter information about the discrepancy or ambiguity related to the particular item. For notes created during CRF data entry, for failed validation checks, the system will automatically include the validation error message in the Discrepancy Note.

Discrepancy types include: Annotation, Failed Validation Check, Query and Reason for Change.

Note: When adding a new discrepancy note to a form, be sure to submit the form before doing anything else, otherwise, your discrepancy note may not be saved to the database.

Viewing Discrepancy Notes
You may view existing discrepancy notes by clicking on any red flag in a form or by clicking the View icon on the Notes & Discrepancy page. Each record in the discrepancy manager will have a View icon, which will open a View Discrepancy Note popup for the note.

The 'View Discrepancy Note' popup will show full information on all the parent and child notes in that thread. It includes (group as shown below):

Entity Type, Subject, Event, Event Date, CRF, Entity Name (if Entity Name is a CRF item it should link to metadata dictionary item).

The Note Description (title), Last Updated Date, Last Updated By, Type, Resolution Status, # of Notes.

If a Parent Note exists: Description, Date Created, Created By, Type, Detailed Note.

For each Child Note: Description, Date Created, Created By, Type, Detailed Note.

There is also an Audit Trail at the bottom of the Note's Description.

Updating and Resolving Discrepancy Notes
Resolving a Discrepancy Note merely means adding a child note with a Resolution Status of 'Closed'. You may do this by clicking on the Reply to thread link down to the right of the popup window.

Resolution Statuses include: New, Updated, Resolution Proposed, Closed and Not Applicable.

Note: You may get an error such as "You may not perform administrative editing on this CRF, so you cannot resolve this note." This will occur if the CRF has not been completed by the user doing data entry, or because your role does not have sufficient privileges to perform administrative editing on this CRF. Please have the user mark the CRF complete to be able to resolve the note.

Once a note has been closed, you may not add additional child notes to that parent note. However, you may create a new parent Discrepancy Note for the same item, allowing you to re-open an old issue as necessary or open a new, unrelated issue on the same data element. Therefore you can click on the Begin New Thread link down left of the popup window.
